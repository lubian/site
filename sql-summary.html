SQL summary

Date Tue 15 March 2016 Category Data Mining Tags SQL

SQL is a widely used language to manipulate relational database. In distributed system, there is also an SQL like language Hive.

A data scientist's daily work may be mainly with R and Python to build statistical models, thus much less with SQL for query operations. Although SQL is a simple and easy-to-apply language, we may somehow forget the language because of lack of practice after some time.
I tried to summarize this language so as to be able to pick up the language quickly whenever needed.

Filter: A basic pattern is as follows

SELECT colnames 
FROM tablename
WHERE condition_on_colvalues

it is possible to add restrictive word after SELECT to specify the certain restrictions: such as

SELECT DISTINCT
SELECT TOP n
SELECT TOP n PERCENT

it is also possible to used combined conditions in WHERE clause, such as

WHERE condition_A AND condition_B  /* or */ /* condition could use operators such as =,>,<,etc */
WHERE condition_A OR condition_B  /* and */
WHERE colelement IN setB  /* categorical value belonging to a set */
WHERE colvalue BETWEEN value1 AND value2 /* numerical value in an closed interval */
WHERE colelement LIKE wildcard_character /* string pattern matching */
WHERE NOT condition /* negative condition */

Sort: to present results in ordered way, we can use ORDER BY

Joins: Sometimes we need to work on the results of joined tables, think of two sets A,B, there are differents possiblities: intersection, union,complement and combinations of the above three.

SELECT colnames 
FROM table1
INNER JOIN table2  /* there exists also LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN,  */
ON table1.key=table2.key

Filter in aggregated results: we using functions such as GROUP_BY, COUNT(), the WHERE clause is no more usable, we must use HAVING instead.

SELECT table2.col1
FROM (table2 
INNER JOIN table1
ON table1.key=table2.key)
GROUP BY table2.col1
HAVING COUNT(table2.col2)>=n

Window Function: sometimes, we need to do operations in a subset, such as the subgroup after GROUP BY, we can use OVER() and PARTITION BY 

SELECT table2.col1, COUNT(table2.col2) OVER (PARTITION BY table2.col1)
FROM (table2 
INNER JOIN table1
ON table1.key=table2.key)

This summary does not includ all the SQL usecases, however, it is sufficient to revive the memory.
